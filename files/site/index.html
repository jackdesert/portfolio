Flywheel App API
My role on the Flywheel server team included making portions of the app geo-aware. For example, what fare structures get displayed to a user based on where in the world the Flywheel app is opened. My work also provided the back-end logic for displaying display special alerts when a livery vehicle was substituted for a regular taxi during peak usage times.

JackListed.com (work-in-progress, started Dec 2013)
This site is currently being built in my spare time as a platform of information sharing among job seekers. The barrier to entry has been kept just millimeters high by not requiring a login at all. Yet there are several methods in place to prevent spam and clutter.
  • Rails 3.2 / Postgres / Nginx on Rackspace
  • Foundation 5 for responsive CSS
  • Haml / Sass for clean views
  • Rspec/RR/Shoulda/FactoryGirl for testing
  • Capistrano 3 for deployment
  • No login required for contributing
  • Reponsive, clean, interaction design requires few mouseclicks
  • Each city has its own subdomain
  • Root path takes you to whatever city you browsed last
  • Requires new URIs to have an http status of 200 (reduces spam and typos)
  • Requires new URIs to have a unique domain name (reduces spam and clutter)
  • Requires new URIs to contain certain keywords or else awaits moderator approval (reduces spam)
